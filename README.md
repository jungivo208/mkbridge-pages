# mkbridge-pages

一个用于归档和发布多个独立 HTML 页面的轻量仓库。

## 项目简介

`mkbridge-pages` 旨在提供一种简洁、可复用的方式，将多个独立的 HTML 页面集中存储在同一个仓库中，并通过 GitHub Pages 或其他静态托管服务进行发布。本仓库不针对任何特定域名或网站，仅作为通用页面存档与展示的载体。

## 目录结构

```
.
├── index.html          # 仓库首页（可选）
├── pages/              # 存放所有独立 HTML 页面的目录
│   ├── example1.html   # 示例页面1
│   ├── example2.html   # 示例页面2
│   └── ...
├── assets/             # 公共资源文件（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md           # 本文件
└── .gitignore          # Git 忽略规则
```

> 实际使用时可根据需求调整目录结构，但建议保持 `pages/` 目录用于存放主要页面文件。

## 页面归档说明

- 所有页面文件均为独立、自包含的 HTML，可各自引用 `assets/` 下的公共资源。
- 页面命名建议使用小写字母、数字和连字符，避免特殊字符与空格。
- 每个页面应包含完整的 HTML 结构（`<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` 等），确保可直接在浏览器中打开。
- 如需引用外部资源（如字体库、第三方脚本），请使用通用 CDN 或本地存放，避免硬编码特定域名。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的导航链接（如有）。
- 更新页面：直接修改对应 HTML 文件内容，保持版本历史清晰。
- 删除页面：从 `pages/` 目录移除文件，并清理相关引用。
- 公共资源：修改 `assets/` 下的文件时，请确保不影响其他页面的正常使用。
- 提交信息建议简明扼要，例如“新增页面：xxx”、“更新页面：xxx 样式调整”。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下创建或放置你的 HTML 页面。
3. 根据需要修改 `assets/` 中的公共资源。
4. 提交并推送至 GitHub，启用 GitHub Pages 功能（选择主分支或 `docs/` 目录）。
5. 访问对应的 Pages 地址即可浏览所有归档页面。

## 注意事项

- 本仓库仅用于存放静态 HTML 页面，不包含后端逻辑或动态内容。
- 请勿将敏感信息（如密码、私钥）直接写入页面源码中。
- 若页面数量较多，建议在 `index.html` 中添加目录或搜索功能以方便导航。
